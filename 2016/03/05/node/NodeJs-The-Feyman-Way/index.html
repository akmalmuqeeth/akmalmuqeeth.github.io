Understand the basics of NodeJs The Feynman Way

2016-03-05
NodeJS, programming

Understand the basics of NodeJS using the Feynman Technique(Whats that? Read here).

1. Concept

A simple google search for NodeJS would return:

  • NodeJS is a software platform which can run programs written in javascript.
  • Its an open-source, cross-platform runtime environment for developing server-side Web applications.
  • The JavaScript is executed by the V8, a JavaScript engine developed by Google which is used in Chrome browser.
  • It uses a JavaScript API to access the network and file system.
  • NodeJS uses asynchronous programming and has an evented I/O model.
  • It is singlethreaded and lock free. This is what makes it really fast.

2. Explain what you know

How much of the above do I really understand? I decided to list down all that I know which would help in identifying gaps in my knowledge.

NodeJS is a software platform not a framework

NodeJs is NOT a framework. It is NOT a collection of libraries. It does NOT provide any scaffolding to create an application.

NodeJs is a platform. A platform on which a software program can execute.

A program is written in Javascript and can be run using NodeJS.

NodeJS provides APIs to access platform resources

To a certain extent, NodeJS is to Javascript, what a JVM is to Java.

A Cross-platform runtime environment which uses Chrome’s V8 Engine :

NodeJS uses Chrome’s V8 engine to compile and execute Javascript.

Node.js is cross-platform meaning it works on Windows, OSX and Linux. If you write your NodeJS code on a Mac OSX it should work cleanly without the need for compilation.

Evented I/O model :

A browser has an event loop. The event loop places items on the queue. I am not very sure how this is achieved by NodeJS though. 

--gap in knowledge identified--

NodeJS Is Single threaded, which makes it really fast :

Single threaded would mean that each incoming request would be handled by the same thread. There is no thread pool. In that sense though, shouldnt t make NodeJS susceptible and slow. Hmmm!

I know what a single threaded system is. I dont really know how it makes NodeJS fast.

An incoming request to a NodeJS server will trigger a Javascript function. Let’s call this an ‘event’ function. The event function does a little bit of work and in turn triggers other (asynchronous) tasks . These other tasks could be connecting to a database or pulling content from another server.

Once these tasks have been kicked off, lets call them ‘Task A’ and ‘Task B’, the original ‘event’ function finishes and Node.js goes back to sleep. When ‘Task A’ or ‘Task B’ are completed, they get back with a response to our NodeJS application and that triggers another ‘event’ function or a callback function.

So to sum it up, NodeJs responds to a event, does some processing and triggers other events. In the meantime it is free to respond to other incoming events. This is why Node.js does such a great job managing thousands of simultaneous connections.

3. Pinpointing (Bridge the knowledge gap)

What does it mean that NodeJS has an evented I/O model?

Heres a good resource I found:

“When creating a web server, there are basically two choices when dealing with multiple concurrent connection requests. The first, which is the more traditional route taken by web servers such as Apache, is to use threads to handle incoming connection requests. The other method, the one taken by Node and some extremely fast modern servers such as Nginx and Thin, is to use a single non-blocking thread with an event loop.

This is where the decision to use JavaScript really shines, since JavaScript was designed to be used in a single threaded event loop-based environment: the browser. JavaScript’s ability to pass around closures makes event-based programming dead simple. You basically just call a function to perform some type of I/O and pass it a callback function and JavaScript automatically creates a closure, making sure that the correct state is preserved even after the calling function has long since gone out of scope.”

4. Analogy

As a webserver, think of NodeJS as an air traffic controller.

Air traffic controllers direct aircrafts efficiently to minimize delays. They manage the flow of aircraft into and out of the airport airspace, guide pilots during takeoff and landing, and monitor aircraft, as they travel through the skies. They guide one pilot and go back to waiting for the next one. When the previous pilot is about to land, he/she contacts the ATC again and gets the clearoff.

NodeJS works in a very similar fashion. It waits for incoming requests, does some minimal processing and triggers other tasks. It doesnt wait for all these other tasks to complete. Infact the tasks get back to NodeJS when they are complete. In the meantime NodeJS is free handle other requests

5. Simplify

NodeJs is a platform which enables you to create web applications using Javascript.
Internally, it uses Google Chrome’s V8 engine to execute the javascript that you wrote.
Its designed in a way to make I/O operations super-fast.
