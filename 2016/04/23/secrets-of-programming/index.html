Best kept secrets of programmers

2016-04-23
point by point, programming

This is my collection of quotes on how to be a better programmer.

  1. Know the concepts. Solving a problem via memory or pattern recognition is much faster than solving it by reason alone.

  2. Know the tools. This is not an end in itself, but a way to maintain “flow” while programming. People who learn their tools, practice using their tools, and automate things that the tools can’t do by themselves can easily be several times as productive as those who do none of those things.

  3. Manage time. If you want to write code, write code. If you want to review a bunch of patches, review a bunch of patches. Don’t try to do all three together. Get your mind set to do one thing, then do that thing for a good block of time before you switch to doing something else.

  4. Reuse everything. Reuse ideas. Reuse code.

  5. If you can’t explain it to a non-programmer, you might be over-complicating or over-optimizing.

  6. Don’t show off by writing “compact code”

  7. It is much better to write something “right” than make it fast and buggy. Whenever you have to hurry, simply don’t, you always have all the time in the world.

    If they say “we don’t have more money, hurry up”, then reduce the scope, remove features, do less, use agile methodologies. Never hurry up.

  8. Unit testing is a lot more effective than traditional QA. Use both at the beginning

  9. You are not being paid to write code, you are paid to write functionality. Code is just cost. There is cost for creating it, cost for debugging it, cost for auditing it, cost for storing it, cost for maintaining it, etc.

  10. The GIGO effect: Garbage in, garbage out. Make sure the requirements are right.

  11. Programming is, ultimately, self-taught. If you’re not spending time outside of class doing this work, you’re memorizing trivia, not learning to program.

  12. The audience for computer code is other human beings, in particular, you six months from now. Be considerate to these people.

  13. Coding is easy, debugging is hard. Code to make debugging trivial. That can be test cases, source control, smaller and well-defined functions….

  14. The best code is code that’s never written because you were able to come up with a non-coding solution or reuse existing code. Remember that as a programmer, you’re hired to solve problems, not write code.

  15. Do not worry about hype. Keep doing your thing, keep learning what you were learning, and move on. Pay attention to it only if you have a genuine interest, or if you feel that it could bring you some benefit in the medium or long run.

  16. Learn at least one new programming language every year.

  17. Read at least 6 books per year.

  18. Do not take a job just for the money.

  19. When learning something new, fight complexity intelligently. Use Feynman technique to better understand complex concepts

  20. As cool as it might sound, do NOT reinvent the wheel. Do a thorough research before you start writing an API from scratch.
