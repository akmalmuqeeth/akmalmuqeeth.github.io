Point by Point - Cordova and Ionic

2016-04-14
Ionic, point by point, programming

Cordova

  1. Apache Cordova enables programmers to build applications for mobile devices using CSS3, HTML5, and JavaScript

  2. Just straight HTML,CSS and Javascript, no platform-specific ‘native’ APIs like those in Android, iOS, or Windows Phone

  3. Cordova applications execute within wrappers targeted to each platform, and rely on standards-compliant API bindings to access each device’s capabilities such as sensors, data, network status, etc.

  4. The resulting applications are hybrid, meaning that they are neither truly native mobile application (because all layout rendering is done via Web views instead of the platform’s native UI framework) nor purely Web-based (because they are not just Web apps, but are packaged as apps for distribution and have access to native device APIs).

Ionic

  1. Ionic is an open source framework which makes it easier to make hybrid mobile apps using Cordova.

  2. Ionic is built on top of AngularJS

  3. Conceptually you could almost think of Ionic as ‘Bootstrap for Mobile’.

  4. Like Bootstrap, Ionic provides a framework, out of the box JavaScript components and CSS, it provides developers that solid foundation upon which to build their apps.

  5. At its core, an Ionic application, is just a web page running in an native app shell! That means we can use any kind of HTML, CSS, and Javascript we want. The only difference is, instead of creating a website that others will link to, we are building a self-contained application experience.

  6. Install globally: npm install -g ionic

  7. Ionic comes with three starter templates - ‘blank’, ‘tabs’ and ‘sidemenu’

  8. Choose your starter template and create an app - ionic start myApp tabs

  9. An application can be run in three ways in ionic:

    • In the browser , navigate to your ionic application and run - ionic serve
    • In the native emulator. First add the platform ionic platform ios then run ionic emulate ios
    • On actual device, ionic run ios

  10. Ionic uses Angular’s UI Router to handle navigation between different application states

  11. Each page in the application is configured as a UI Router state
